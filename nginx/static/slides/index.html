01 · Contexto

El problema

  • El precio del alquiler en España ha subido más de un 40 % en la última década
  • Buscar habitación compartida sigue siendo un proceso opaco y frustrante
  • Los anuncios sin verificar generan desconfianza en arrendadores e inquilinos
  • No existe una plataforma pensada para la convivencia a medio y largo plazo
  • El fraude en plataformas de alquiler es uno de los problemas más denunciados
PlataformaLimitación
AirbnbSolo turismo / estancias cortas
IdealistaOrientado a inmuebles completos
BadiAlcance reducido, sin moderación
SpotahomeInternacional, comisiones altas
Ninguna plataforma combina moderación de anuncios, app móvil nativa y gestión de convivencia en un solo producto.

CREATE UNIQUE INDEX uq_property_main

06 · Seguridad

Autenticación JWT

POST /v1/auth/login
↓
Access Token (1 h) + Refresh Token (24 h)
↓
Authorization: Bearer <token>
↓
JwtAuthenticationFilter — OncePerRequestFilter
↓
validateToken → loadUser → SecurityContext
↓
@PreAuthorize("hasRole('ADMIN')")
String token = header.substring(7);
if (jwtProvider.validateToken(token)) {
  String username =
    jwtProvider.getUsername(token);
  UserEntity user = userRepository
    .findByUsername(username)
    .orElse(null);
  var auth =
    new UsernamePasswordAuthenticationToken(
      user, null,
      List.of(new SimpleGrantedAuthority(
        "ROLE_" + user.getRole())));
  SecurityContextHolder.getContext()
    .setAuthentication(auth);
}
  • Refresh tokens persistidos y revocables en BBDD
  • Logout invalida el refresh token activo
  • CORS de /v1/admin/** restringido a admin.habitame.xyz
07 · Funcionalidad clave

Moderación de anuncios

Arrendador publica propiedad / habitación
↓
IN_REVIEW — invisible al catálogo público
↓
Admin revisa desde el panel de administración
↓                   ↓
ACTIVE — visible
REJECTED + comentario
↓
Arrendador ve el feedback → corrige → reenvía
@Transactional
public PropertyReviewResponse resolveReview(
    Integer propertyId,
    PropertyReviewDecisionRequest req) {

  // Rechazo obliga a justificarlo
  if (req.status() == REJECTED
      && isBlank(req.comment()))
    throw new IllegalArgumentException(
      "Se requiere comentario al rechazar");

  review.setStatus(req.status());
  review.setAdmin(
    SecurityUtils.getCurrentUser());
  review.setReviewedAt(LocalDateTime.now());

  property.setStatus(
    req.status() == APPROVED
      ? ACTIVE : IN_REVIEW);

  return mapper.toResponse(
    reviewRepository.save(review));
}

Future<Response> _handleResponse(
    Response response,
    Future<Response> Function(String) retry
) async {
  if (response.statusCode == 401) {
    final token =
      await _authService.refreshToken();
    if (token != null)
      return retry(token); // reintenta
  }
  if (response.statusCode >= 400)
    throw ApiResponseError
      .fromJson(jsonDecode(response.body));
  return response;
}
El usuario nunca ve una pantalla de sesión caducada — la renovación es completamente transparente.

12 · Landing web

Landing web Angular SSR

  • Server-Side Rendering con Angular Universal — HTML generado en el servidor antes de enviarse al navegador
  • Indexación correcta por buscadores — SEO y Open Graph sin necesidad de JavaScript
  • Tiempo hasta primer pintado (FCP) significativamente menor que una SPA clásica
  • Consume la misma API pública que la app móvil (/v1/public/**)
  • Catálogo actualizado de propiedades disponibles en tiempo real
  • CTA para descargar la app móvil
